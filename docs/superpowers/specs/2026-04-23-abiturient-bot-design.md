# Дизайн: Telegram-бот для абитуриентов колледжа

**Дата:** 2026-04-23
**Стек:** Python, aiogram 3.x, SQLite, Claude Haiku API

---

## 1. Цель

Telegram-бот для абитуриентов колледжа, который:
- Отвечает на вопросы по специальностям, грантам, общежитию
- Ведёт FAQ
- Проводит профориентационный тест с ИИ-рекомендациями
- Принимает записи на экскурсии и дни открытых дверей
- Позволяет администратору управлять датами мероприятий и контентом

## 3. Функциональность

### 3.1 Кнопочное меню
Основная навигация через inline-кнопки. Каждый раздел (специальности, гранты, общежитие) подтягивает текст из базы знаний SQLite.

### 3.2 Свободный ввод текста
Если абитуриент пишет сообщение не через кнопки:
1. Поиск релевантного текста в базе знаний по ключевым словам
2. Релевантный кусок + вопрос отправляются в Claude Haiku
3. Ответ возвращается абитуриенту

Если релевантный текст не найден — бот предлагает написать администратору.

### 3.3 FAQ
- Список вопрос/ответ, хранится в SQLite
- Абитуриент выбирает вопрос из списка кнопок
- Администратор управляет через `/addfaq` и `/delfaq`

### 3.4 Профориентационный тест
- 15 вопросов с вариантами ответов, подаются по одному
- Ответы накапливаются в памяти сессии
- После последнего вопроса: все ответы + список специальностей колледжа → Claude Haiku
- Вывод: развёрнутый отчёт с анализом интересов, сильных сторон и рекомендацией специальностей

### 3.5 Запись на экскурсии / дни открытых дверей
**Администратор** добавляет даты через `/adddate`.
**Абитуриент** видит список доступных дат с оставшимися местами, выбирает одну, вводит имя и телефон.
После записи:
- Место в базе уменьшается на 1
- Администратор получает уведомление в Telegram
- Абитуриент получает подтверждение

### 3.6 База знаний
- Источники: сайт колледжа (BeautifulSoup) + документы (PDF/Word/Excel)
- Парсинг сайта: вручную `/update` или автоматически раз в неделю
- Загрузка документов: администратор кладёт файлы в папку `docs/` и запускает `/reload_docs`
- Хранение: SQLite, разбивка по секциям (специальности, гранты, общежитие, прочее)

## 4. База данных

```sql
-- База знаний
knowledge_base (
    id INTEGER PRIMARY KEY,
    source TEXT,           -- 'site' или имя файла
    section TEXT,          -- 'гранты', 'специальности', 'общежитие', 'прочее'
    content TEXT,
    updated_at TIMESTAMP
)

-- FAQ
faq (
    id INTEGER PRIMARY KEY,
    question TEXT,
    answer TEXT,
    created_at TIMESTAMP
)

-- Даты мероприятий
excursion_dates (
    id INTEGER PRIMARY KEY,
    date DATE,
    time TIME,
    type TEXT,             -- 'экскурсия' или 'день открытых дверей'
    max_slots INTEGER,
    current_slots INTEGER,
    is_active BOOLEAN
)

-- Заявки
excursion_requests (
    id INTEGER PRIMARY KEY,
    excursion_date_id INTEGER REFERENCES excursion_dates(id),
    name TEXT,
    phone TEXT,
    telegram_user_id INTEGER,
    created_at TIMESTAMP
)
```

## 5. Команды администратора

| Команда | Описание |
|---|---|
| `/update` | Перепарсить сайт колледжа |
| `/reload_docs` | Загрузить документы из папки `docs/` |
| `/addfaq` | Добавить вопрос в FAQ |
| `/delfaq` | Удалить вопрос из FAQ |
| `/adddate` | Добавить дату мероприятия |
| `/deldate` | Удалить дату мероприятия |
| `/dates` | Посмотреть все даты и количество записавшихся |
| `/stats` | Статистика бота (пользователи, запросы) |

Команды доступны только пользователям с `ADMIN_TELEGRAM_ID` из `.env`.

---

## 6. Обработка ошибок

| Ситуация | Поведение бота |
|---|---|
| Claude API недоступен | "Не могу ответить сейчас, напишите администратору: @username" |
| Ответ не найден в базе знаний | Предлагает написать администратору |
| Дата мероприятия заполнена | Сообщает что мест нет, предлагает другие даты |
| Документ не читается | Пропускается, записывается в лог |

## 8. Деплой

- Платформа: Railway или VPS (Timeweb)
- Режим: long polling (проще для старта) или webhook
- Стоимость: ~$3-5/месяц за сервер + ~$4-5/месяц за Claude API при 100 пользователях в день
